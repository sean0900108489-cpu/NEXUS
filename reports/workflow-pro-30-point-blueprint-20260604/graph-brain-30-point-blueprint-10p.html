Page 02

Hidden-Answer Test Principle

02 / 10

實測不能直接給 Brain 標準拓撲。它應該只看到自然語言需求，例如「我想上傳一張圖，讓兩個不同角色的模型分析後給我最後答案」。拓撲答案只存在 evaluator 的 hidden rubric 裡。

Brain sees

意圖、素材、目標、限制、想要輸出。

Brain infers

節點類型、順序、角色、分支、輸出、缺口。

Evaluator scores

是否覆蓋能力、JSON 是否合法、是否可 append、是否誠實。

Phase 1 - Scan And Contract Readiness

05 / 10

第一階段不急著寫 UI，要確認沒有重複 converter、沒有新 schema 分裂、沒有繞過 validator。

Round 1

掃 Graph Brain panel、RuntimeLite append group、validator、bridge、model registry、route pattern、auth/session gate。

Round 2

定義 reusable entry points：context pack builder、Brain request payload、Brain result parser、append result feedback。

Exit Gate

所有 Graph Brain 生成工作流都必須回到 `nexus.workflow.v1` 與 canonical runtime bridge。

Phase 2 - Brain API And Context Pack

06 / 10

Brain 不能只讀使用者一句話。它必須知道目前畫布裡有哪些 group、哪些 node types 可用、哪些能力缺失、哪些 runtime evidence 可用。

Round 3

建立 Graph Brain API/service boundary，使用安全的後端 key，不把憑證送到前端。

Round 4

建立 canvas context pack：nodes、edges、groups、capabilities、missing features、artifact/run evidence。

Round 5

建立 two-step LLM protocol：Intent Architect → JSON Contract Compiler。
