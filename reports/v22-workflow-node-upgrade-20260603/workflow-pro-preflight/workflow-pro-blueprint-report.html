Workflow Pro 要成為專業工作流設計艙，不是 Graph 的複製品。

這份藍圖規劃第三個 workspace mode：`PANELS | GRAPH | WORKFLOW PRO`。它把 workflow 設計、大腦分析、證據時間線、proposal diff、檔案節點與轉檔槽整合成一個深度開發任務的前置規格。

左上角新增 Workflow Pro。

現有 `WorkspaceViewMode` 只有 panels/graph，因此第一輪工程要從型別與持久化開始，而不是只改按鈕文字。

Type layer

`WorkspaceViewMode` 擴成 `"panels" | "graph" | "workflow-pro"`，並確認 import/sanitize 舊快照時不壞。

Store layer

`setViewMode("workflow-pro")` 同步 global store 與 active workspace settings，和 panels/graph 同等級。

UI layer

左上 workspace card 變成三段式頁籤。Workflow Pro label 必須穩定放入現有寬度，必要時縮成 `PRO` + tooltip。

Workflow Pro surface 的內部結構。

參考 Graph 的基礎互動，但不照抄。Workflow Pro 的主職責是 design intelligence cockpit。

內部分頁

  • Design：contract-aware workflow builder，顯示 workflow 目的、節點語義、packet flow。
  • Brain：Workflow Brain 摘要、理解狀態、問題、風險、缺少能力。
  • Evidence：工程圖 5 的 Evidence Timeline Cockpit，專看 run evidence。
  • Proposal Diff：工程圖 6 的 Proposal Diff War Room，專看大腦改版建議。
  • Files：檔案節點、raw artifact、compiler output、packet attachment。
  • Settings：Brain boot prompt、context pack source、developer handoff toggles。

關鍵取捨

Evidence 和 Proposal Diff 都很強，但放在同一屏會擠。Workflow Pro 應讓它們共用同一個 analysis bay，用 segmented control 切換。

Evidence Proposal Diff One bay No crowding

Evidence Timeline View

專門回答「這次 workflow 實際怎麼跑」。顯示節點狀態、時間線、artifact、錯誤、耗時與大腦診斷；適合 run 完後檢查瓶頸與 packet mismatch。

Run evidence Artifact trail Diagnosis

Proposal Diff View

專門回答「大腦建議怎麼改」。比較 current workflow 與 optimized workflow，列出變更理由、風險、缺少能力與 suggested Codex tasks；第一版只產生可審核 proposal，不自動套用。

新增 file-carrying node。

這個節點讓文字通過時把檔案一起夾帶，並預留轉檔/解壓/提取/編譯能力。

input.file
  raw file artifact
  noop compiler now
  compiler slot later
  packet attachments
  downstream LLM/image/video

future compilers
  compiler.zip.extract
  compiler.pdf.ocr
  compiler.image.vision
  compiler.video.frames
  compiler.audio.transcript

為什麼這很重要

語言模型原本不能直接處理的檔案，例如 zip，可以先保存 raw artifact，經 compiler 轉成 LLM 可讀的文字、索引、檔案樹或引用，再夾在 ContextPacket 上往下游傳。
