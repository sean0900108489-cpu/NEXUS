03 / Product Boundary

Workflow Pro 的任務是理解、設計、批判與再生成

Graph 是可執行節點畫布；Workflow Pro 是專業設計與智能分析層。 它要能看見工作流順序、串聯、並聯、fallback、缺失能力、節點理由、模型設定、 檔案編譯策略和輸出 artifact policy。這是讓 Workflow Brain 從一開始就知道整個 workflow 在幹嘛的前提。

04 / Canonical JSON

一個 JSON 文件生成、解釋、驗證與優化 workflow

nexus.workflow.v1 是這次最重要的地基。它不是 export snapshot， 也不是 runtimeLite 執行狀態，而是 brain-readable workflow design。 每個 node 和 edge 都要寫明存在原因與資料流契約。 

Workspace snapshot = recovery and portability.
Runtime Lite = executable run state.
nexus.workflow.v1 = brain-readable workflow design.
05 / Workflow Brain

大腦不是聊天框，是讀懂全圖的監督模型

Workflow Brain 在每次開啟時都應讀入 workflow contract、capability inventory、 compiler registry、artifact policy、runtime evidence 和 known missing capabilities。 它必須先解釋這個 workflow 想完成什麼，再指出順序、風險、弱提示詞、模型設定錯配、 缺失能力，最後才提出可執行的優化版 workflow。

人類可讀

它要用清楚語言解釋目前流程在幹嘛、哪裡可以更好、哪些問題需要 Sean 決策。

機器可接手

它要輸出 machine-readable proposals，讓 Codex 可以把缺失能力轉成下一批升級任務。

07 / Files And Compilers

檔案節點走 raw artifact -> compiler -> compiled artifact

檔案節點不是普通附件。它要能把原始檔案帶著走，並且在通過節點時掛上 compiler metadata。 不需要轉換的檔案也走 no-op compiler。未來 zip、pdf、影片、圖片、向量化、摘要、抽幀或 OCR， 都能接到同一條線路，不必打散整個訊息系統。 

raw file artifact
-> compiler metadata
-> compiled artifact
-> ContextPacket attachment reference
